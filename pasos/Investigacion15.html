# 00 / tl;dr

Tres conclusiones que mueven la aguja — por qué Naeth tiene sentido

ganador[OK]

doobidoo/mcp-memory-service

Es el único proyecto del Top que combina los tres factores críticos: (1) compatibilidad explícita con claude.ai (Remote MCP via HTTPS) y con Claude Code (hooks + Code Execution API desde v8.19.0+), (2) local-first real con SQLite-vec por defecto y embeddings ONNX/sentence-transformers sin API keys, y (3) cobertura amplia (24 herramientas MCP, KG, consolidación autónoma, temporal queries, OAuth 2.1). Apache-2.0/MIT, releases semanales, 13+ clientes MCP.

benchmark[##]

Hindsight (Vectorize) lidera

Paper "Hindsight is 20/20: Building Agent Memory that Retains, Recalls, and Reflects" (arXiv:2512.12818, dic. 2025): 91.4% LongMemEval con Gemini-3 Pro, 83.6% con OSS-20B local. MIT, MCP server integrado, embedded Postgres, soporta Ollama. Pero el headline 91.4% requiere LLM frontera — no es el score del sistema de memoria aislado.

alerta[!!]

El espacio comercial es minado

Mem0 hizo sunset de OpenMemory, Zep Community Edition fue deprecada abril 2025 (solo Graphiti queda OSS), Supermemory bloquea self-host detrás de contrato enterprise, y Cipher (campfirein) pasó a Elastic License 2.0 y fue renombrado a ByteRover CLI. El hueco "memoria personal portable para chat clients" sigue abierto.

# 04 / hallazgos transversales

Cinco patrones del ecosistema — aplicables a Naeth

01

LongMemEval está siendo manipulado o mal-reportado

Casi nadie publica el modo (R@5 vs R@10 vs E2E QA), el LLM lector, ni la configuración (oráculo de sesiones vs corpus completo). El paper original (Wu et al., ICLR 2025, arXiv:2410.10813) define el benchmark como QA end-to-end; las cifras altas (>90%) tienden a ser retrieval-only sobre haystacks pequeños. Tratar toda cifra > 85% como "claim del proyecto" hasta replicación independiente.

02

Solo 2 declaran soporte explícito de claude.ai Remote MCP

doobidoo/mcp-memory-service (lo lista textualmente en su README junto a ChatGPT Developer Mode) y Anthropic Memory MCP oficial (porque claude.ai lo soporta nativamente). El resto soporta sólo Claude Code/Desktop vía stdio o HTTP local.

03

El vendor lock-in del espacio comercial es mayor de lo que parece

OpenMemory (mem0) fue sunset; Zep Community Edition deprecada abril 2025; Supermemory requiere enterprise agreement; Cipher (Byterover) pasó a Elastic License 2.0 y fue renombrado a ByteRover CLI. El hueco "memoria personal portable para chat clients" sigue abierto.

04

MemPalace amerita inclusión con asterisco grande

19-42k stars cuestionados (audits independientes). El 96.6% R@5 es esencialmente el score de ChromaDB+all-MiniLM-L6-v2 sin que el código palace intervenga (verificado en Issue #39, reproducción en M2 Ultra). El 100% hybrid requiere Claude Haiku pagado. No usar como base técnica; sí como caso de estudio de cómo NO comunicar benchmarks.

05

El verdadero state-of-the-art académico es Hindsight + EverMemOS

Ambos con preprints reproducibles. Hindsight tiene la mejor combinación de benchmark + licencia MIT + integración Claude Code + soporte local (Ollama + embedded Postgres). EverMemOS reporta 83% LME-S / 93% LoCoMo pero sin venue peer-reviewed.

# 07 / caveats

Lo que hay que leer con escepticismo

// benchmarks
Casi ninguna cifra > 85% en este espacio está replicada independientemente. El paper LongMemEval original (Wu et al., ICLR 2025, arXiv:2410.10813) define el benchmark como QA end-to-end (retrieval + LLM + judge), pero proyectos reportan a menudo retrieval-only R@5/R@10 sin clarificarlo. Hindsight (replicado por Virginia Tech's Sanghani Center y Andrew Neeser / Washington Post) y Graphiti (paper arXiv:2501.13956) son las excepciones más sólidas. MemPalace es el ejemplo paradigmático de manipulación.
// licencias
Verificar manualmente: mcp-memory-service (Apache vs MIT inconsistencia README), Basic Memory (probable AGPL → fricción para SaaS derivativo), MARM-Systems (no explícita), Memori (no explícita). Antes de derivar código, leer el archivo LICENSE del repo.
// ecosistema
La capa de memoria persistente para LLMs está en transición agresiva. OpenMemory (mem0) sunset, Zep CE deprecada (abril 2025), Cipher → Elastic License 2.0 → ByteRover CLI (abril 2026), MemPalace en controversia. Cualquier ranking debe re-validarse cada 2-3 meses.
// vendor memory
Anthropic ha lanzado la Memory Tool nativa en claude.ai/Claude Code. Esto es complementario (vendor lock-in implícito) pero el usuario lo descartó correctamente. Naeth debe interoperar con la memoria nativa, no competir — import/export y sync bidireccional con el memory.jsonl del Anthropic Memory MCP.
// stars github
Las cifras de stars no son fiables como métrica de calidad post-MemPalace. Priorizar: frecuencia de commits, contributors únicos, releases en los últimos 90 días, issues respondidos con turnaround < 7 días.
