Desplegar Naeth v1 en finally

Runbook detallado: cifrado LUKS, backups, compose de Postgres + pgvector, y exposición por el túnel enraxk + Cloudflare Access. v1 = el visor re-plataformado a FastAPI + Postgres. Nada se ejecuta sin tu OK, fase a fase; los comandos son ilustrativos, verificar antes.

§00topología

Cómo encaja todo #

navegador (cualquier parte)  ──HTTPS──▶  Cloudflare edge (Access: identidad + 2FA)
                                              │  túnel enraxk (saliente, sin puertos abiertos)
                                              ▼
finally:  cloudflared ──▶ 127.0.0.1:8008  (Naeth API · FastAPI)
                               │  red docker interna (sin puerto al host)
                               ▼
                          Postgres + pgvector  ──▶  pgdata en volumen LUKS
                               ▲
                          worker de embeddings (CPU)
                          assets (PDF/HTML/...) en disco LUKS
  • Postgres no se expone al host: solo red interna del compose.
  • API solo en loopback (127.0.0.1:8008); único acceso externo = el túnel.
  • Datos cifrados (LUKS) y con backup antes de meter nada real.
  • Secretos en .env 600, patrón X-Enraxk-Token.
§01prerequisitos

Cerrar huecos antes de exponer #

El informe de finally dejó tres huecos que condicionan meter datos reales.

1.1 Endurecimiento base

El combo actual (UFW inactivo + CouchDB 5984 público + SSH con contraseña + sin fail2ban) es la mayor exposición.

sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp                  # SSH
sudo ss -tlnp                          # revisar qué hay publicado; CouchDB 5984 NO debe serlo
sudo ufw enable
sudo apt install fail2ban && sudo systemctl enable --now fail2ban
# SSH solo-clave (verifica PRIMERO que tienes llave): PasswordAuthentication no
sudo sshd -t && sudo systemctl reload ssh
docker rm squid-proxy                   # el que estaba en restart loop
cuidado

No deshabilitar PasswordAuthentication sin confirmar acceso por clave en una segunda sesión SSH, o te quedas fuera.

1.2 Cifrado en reposo (q2): volumen LUKS

En vez de re-cifrar todo, un contenedor LUKS dedicado para pgdata + assets (hay 214 GB libres).

sudo lvcreate -L 50G -n naeth-data ubuntu-vg
sudo cryptsetup luksFormat /dev/ubuntu-vg/naeth-data   # AES-256-XTS
sudo cryptsetup open /dev/ubuntu-vg/naeth-data naeth-data
sudo mkfs.ext4 /dev/mapper/naeth-data
sudo mkdir -p /srv/naeth/data && sudo mount /dev/mapper/naeth-data /srv/naeth/data

Apertura en arranque: para homelab, lo simple es desbloqueo manual tras reboot (SSH + cryptsetup open + mount). El compose de Naeth no debe arrancar si el LUKS no está montado.

1.3 Backups (crítico, no hay ninguno)

sudo apt install restic
restic init --repo /ruta/backup/naeth         # RESTIC_PASSWORD en .env 600; destino aparte
# naeth-backup.sh (cron diario):
#  docker exec naeth-db pg_dump -U naeth naeth | restic backup --stdin --stdin-filename naeth.sql
#  restic backup /srv/naeth/data/assets ; restic forget --keep-daily 7 --keep-weekly 4 --prune
probar

Probar un restore antes de confiar. El mismo patrón conviene para el CouchDB del stack viejo, que tampoco tiene backup.

§02el stack

docker-compose #

services:
  db:
    image: pgvector/pgvector:pg16
    shm_size: 8gb                       # crítico para el build paralelo de HNSW
    environment: { POSTGRES_USER: naeth, POSTGRES_DB: naeth }
    volumes:
      - /srv/naeth/data/pgdata:/var/lib/postgresql/data   # LUKS
      - ./db/postgresql.tuning.conf:/etc/postgresql/conf.d/tuning.conf:ro
    networks: [naeth-internal]          # SIN ports: no se expone al host
    healthcheck: { test: ["CMD-SHELL","pg_isready -U naeth"], interval: 10s }
  api:
    build: ./api
    env_file: .env
    ports: ["127.0.0.1:8008:8000"]      # solo loopback; el túnel mapea aquí
    depends_on: { db: { condition: service_healthy } }
    networks: [naeth-internal]
  worker:
    build: ./worker
    env_file: .env
    networks: [naeth-internal]
networks: { naeth-internal: { driver: bridge } }

Tuning de Postgres (62 GB RAM)

shared_buffers = 8GB
effective_cache_size = 24GB
maintenance_work_mem = 2GB             # builds HNSW grandes en RAM (Paso 5)
max_parallel_maintenance_workers = 4
.env (600)

POSTGRES_PASSWORD (nuevo, no reutilizar), NAETH_API_TOKEN (X-Enraxk-Token), RESTIC_PASSWORD, EMBED_MODEL=BAAI/bge-small-en-v1.5. Valores nunca en docs.

§03cloudflare

Exposición (q1) #

El túnel enraxk ya existe; su ingress se gestiona en el dashboard de Zero Trust, no en el server.

1Zero Trust → Networks → Tunnels → enraxk → Public Hostnames → Add: subdomain naeth, domain enraxk.dev, service HTTP → localhost:8008.
2Zero Trust → Access → Applications → Add (Self-hosted): dominio naeth.enraxk.dev, policy Allow solo tu email con 2FA / OTP. Login desde cualquier navegador, sin abrir puertos.
3(Opcional) defensa en profundidad: sesión app-level argon2 en la API.
cero red

No hace falta tocar red en finally: el túnel sale hacia Cloudflare. q1 resuelto.

§04arranque

Migraciones y up #

1Montar el LUKS (cryptsetup open + mount) antes del compose.
2docker compose up -d db y esperar healthy.
3Esquema (Paso 6): CREATE EXTENSION vector; + DDL de memory/relation/attachment/job, la vista memory_current y los índices (con el tuning de build).
4docker compose up -d api worker y smoke test (§05).
§05verificación

Smoke test #

  • docker compose ps healthy; db sin puerto publicado (ss -tlnp no muestra 5432).
  • API local: curl -H "X-Enraxk-Token: ..." http://127.0.0.1:8008/api/status.
  • Alta + recall: escribir una memoria, ver que la búsqueda híbrida responde rápido.
  • Cola: la memoria nueva genera job(embed); el worker la procesa; el desfase baja a 0.
  • Acceso externo: abrir https://naeth.enraxk.dev desde fuera; Access pide 2FA; carga el visor.
  • Backup: ejecutar naeth-backup.sh a mano y probar un restore.
§06checklist

"Producción casera" #

  • UFW activo, CouchDB no público, fail2ban activo, SSH solo-clave.
  • Volumen LUKS montado, pgdata y assets dentro.
  • Backups cifrados (restic) por cron, con restore probado.
  • Postgres sin puerto al host; API solo en loopback.
  • Cloudflare Access con 2FA sobre naeth.enraxk.dev.
  • .env con permisos 600, secretos nuevos.
  • cloudflared actualizado.
§07orden

Ejecución por fases (cada una con tu OK) #

1Endurecimiento base (§1.1): barato, reduce exposición ya.
2LUKS + backups (§1.2-1.3): prerequisito de datos reales.
3Esqueleto del stack (§2): compose + .env + tuning, sin datos.
4DDL del esquema (§4): tablas e índices.
5Construir la API v1 (visor re-plataformado) + worker. Aquí entra el código.
6Exponer (§3) y verificar (§5-6).
